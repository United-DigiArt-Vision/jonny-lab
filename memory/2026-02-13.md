### KRITISCH: xAI API Key Leak + Revoke (01:49)

**Was passiert ist:** xAI hat unseren API Key revoked weil er in einem öffentlichen GitHub Repo gefunden wurde.
**Ursache:** `secrets/accounts.json` wurde mit `git push` ins öffentliche Repo `United-DigiArt-Vision/jonny-lab` hochgeladen.
**Mein Fehler:** Ich hätte secrets/ von Anfang an in .gitignore haben müssen.

**Sofortmaßnahmen:**
1. `secrets/accounts.json` aus Git-Tracking entfernt (`git rm --cached`)
2. `.gitignore` erweitert: `secrets/`, `*.db`, `*.db-wal`, `*.db-shm`, `*.db.lock`, `tools/*/cache/`
3. Research-Caches und SQLite DBs aus Repo entfernt
4. **Pre-Commit Hook installiert** (`.git/hooks/pre-commit`) — blockt xAI, OpenAI, GitHub, Google, AWS Key-Patterns + password/api_key JSON-Patterns
5. Hook getestet und funktioniert

**Noch offen:**
- Dino muss neuen xAI Key generieren auf console.x.ai
- Git History enthält noch den alten Key (revoked, kein akutes Risiko)
- Optional: BFG Repo Cleaner für History-Bereinigung

**Lesson:** JEDES Repo das Secrets enthält braucht von Tag 1: .gitignore für secrets + pre-commit hook. Nie wieder ohne.
