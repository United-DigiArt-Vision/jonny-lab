## 2. ðŸ”’ Deno Sandbox: Hosted Sandboxes with Secret Protection

**Source:** Simon Willison
**URL:** https://simonwillison.net/2026/Feb/3/introducing-deno-sandbox/

**Key Learnings:**
- Deno launched a hosted sandbox product (part of Deno Deploy SaaS)
- Language-agnostic: can use Python or JavaScript client libraries
- Specs: up to 4GB RAM, 2 vCPUs, 10GB ephemeral storage, 30min sessions
- **Brilliant secret handling:** API keys are replaced with placeholders inside sandbox
  - e.g., `DENO_SECRET_PLACEHOLDER_b14043a2f578cba...`
  - Outbound calls go through proxy that swaps placeholders back to real secrets
  - **Result:** Malicious code (prompt injection) can't exfiltrate secrets!
- Network access control: configure allowed domains per sandbox
- Fly.io has similar project called "tokenizer" (https://github.com/superfly/tokenizer)

**Relevance for Us:**
- **Security pattern we should use!** For any AI-driven tool that handles API keys
- Sandboxed execution environment = safe way to run untrusted/AI-generated code
- Could be useful for web app prototyping (spin up sandbox for testing)
- Secret placeholder technique = brilliant solution to prompt injection exfiltration
